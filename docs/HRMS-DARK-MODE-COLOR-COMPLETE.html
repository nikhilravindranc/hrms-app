🌙 HRMS Dark Mode Color System

Complete Dark Palette with Usage & Implementation

Adapted from the EVOQ dark mode system — every blue accent replaced with the HRMS teal palette. All colors documented with WCAG contrast specifications.

🔁 0. What Changed From the CRM Version

This system reuses the CRM's dark mode structure (backgrounds, borders, neutral text tiers) as-is — that part was already color-agnostic. Only the blue-hued accent colors were replaced with HRMS's teal identity. Semantic status colors (success/error/warning) are unchanged since they carry meaning independent of brand color.

Role CRM (Blue) HRMS (Teal) Why
Accent / link hover / highlights #60A5FA #27EAA6 Brand Hover teal — brighter contrast than the blue it replaces (12.6:1 vs 5.6:1)
Primary button background #1D4ED8 #00755A Brand Action teal
Primary button active/pressed #0C2472 #003D2E Brand Depth teal — darkest step in the palette
Info status color #60A5FA #5E93FF Uses the palette's own Purple complementary accent instead of a generic blue, and stays visually distinct from the teal accent used for hover/focus
Backgrounds, borders, neutral text tiers Unchanged — already brand-agnostic No blue hue involved
Success / Error / Warning Unchanged — #10B981 / #EF4444 / #F59E0B Semantic, not brand color

📍 Border Rules:

  • Focused inputs: 2px #27EAA6 (bright teal)

💻 8. CSS Variables & Implementation

Use CSS variables for easy dark mode implementation and swapping. Matches the variables already defined in app/globals.css.

CSS Variable Setup

:root { /* Light Mode - Default */ --bg-primary: #E8EFF6; --bg-surface: #FFFFFF; --text-primary: #0C2472; --text-secondary: #334155; --text-tertiary: #94A3B8; --border-primary: #D4E8E0; --color-primary: #004D43; --color-hover: #27EAA6; } [data-theme="dark"] { /* Dark Mode */ --bg-primary: #0A0A0A; --bg-surface: #18181B; --text-primary: #D4D4D8; --text-secondary: #71717A; --text-tertiary: #9CA3AF; --border-primary: #27272A; --color-primary: #00755A; --color-hover: #27EAA6; --color-active: #003D2E; } /* Usage in components */ body { background-color: var(--bg-primary); color: var(--text-primary); } .card { background-color: var(--bg-surface); border: 1px solid var(--border-primary); } button { background-color: var(--color-primary); } button:hover { background-color: var(--color-hover); }

Tailwind CSS Dark Mode

/* In component - Light mode is default */ <div class="bg-[#E8EFF6] text-[#0C2472] dark:bg-[#0A0A0A] dark:text-[#D4D4D8]"> Content </div> /* Button example */ <button class="bg-[#004D43] hover:bg-[#27EAA6] dark:bg-[#00755A] dark:hover:bg-[#27EAA6] text-white"> Click me </button> /* Form input example */ <input class="bg-white border-[#D4E8E0] text-[#0C2472] dark:bg-[#18181B] dark:border-[#27272A] dark:text-[#D4D4D8] focus:border-[#004D43] dark:focus:border-[#27EAA6]" />

✅ 10. Dark Mode Implementation Checklist

Before launching dark mode, verify:
  • ☐ All text meets 4.5:1 contrast minimum (WCAG AA)
  • ☐ Links are distinguishable from body text
  • ☐ Focused elements have clear focus ring (3px #27EAA6)
  • ☐ Buttons and interactive elements are clearly clickable
  • ☐ Icons are visible and properly colored for dark mode
  • ☐ Form inputs have clear borders and focus states
  • ☐ Tables are readable with proper contrast
  • ☐ Modals stand out against page background
  • ☐ Status colors (success/error/warning/info) are distinguishable from each other and from the teal accent
  • ☐ Hover/focus/active states are all visible
  • ☐ Transitions between light/dark are smooth (150-200ms)
  • ☐ Images don't appear blown-out or washed-out
  • ☐ Color is not the only indicator of state/status
  • ☐ No leftover blue hues (#60A5FA, #1D4ED8, #0C2472-as-accent, #3B82F6) outside of categorical chart colors
